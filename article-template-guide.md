# دليل استخدام قالب المقالات

## نظرة عامة

تم إنشاء قالب احترافي للمقالات (`article-template.html`) يتضمن جميع الميزات المطلوبة لإنشاء مقالات عالية الجودة مع دعم كامل لنظام الإعلانات وتحسين محركات البحث.

## الميزات المتضمنة

### 1. تحسين محركات البحث (SEO)
- **Meta Tags**: عناوين ووصف محسن
- **Open Graph**: دعم مشاركة وسائل التواصل الاجتماعي
- **Schema.org**: بيانات منظمة للمقالات
- **Canonical URLs**: روابط قانونية لتجنب المحتوى المكرر

### 2. التصميم المتجاوب
- **Mobile-First**: مُحسَّن للهواتف المحمولة أولاً
- **Responsive Design**: يتكيف مع جميع أحجام الشاشات
- **Touch-Friendly**: أزرار وعناصر سهلة اللمس

### 3. تجربة المستخدم المحسنة
- **شريط تقدم القراءة**: يظهر مدى تقدم القارئ في المقال
- **أزرار المشاركة**: مشاركة سهلة على وسائل التواصل
- **التنقل بين المقالات**: روابط للمقال السابق والتالي
- **العودة للمدونة**: رابط سريع للعودة لصفحة المدونة

### 4. نظام الإعلانات المدمج
- **إعلان أعلى المقال**: يظهر قبل بداية المحتوى
- **إعلان وسط المقال**: يظهر في منتصف المحتوى
- **إعلان أسفل المقال**: يظهر بعد انتهاء المحتوى
- **إدارة تلقائية**: يتم إدراج الإعلانات تلقائياً

## كيفية إنشاء مقال جديد

### الخطوة 1: نسخ القالب
```bash
cp article-template.html new-article.html
```

### الخطوة 2: تحديث المعلومات الأساسية

#### في قسم `<head>`:
```html
<!-- استبدل هذه القيم -->
<meta name="description" content="وصف المقال الجديد">
<meta name="keywords" content="الكلمات, المفتاحية, للمقال">
<meta property="og:title" content="عنوان المقال - الجمال">
<meta property="og:description" content="وصف المقال">
<meta property="og:url" content="https://www.algmmal.com/new-article.html">
<title>عنوان المقال - الجمال</title>
```

#### في Schema.org:
```json
{
    "@context": "https://schema.org",
    "@type": "BlogPosting",
    "headline": "عنوان المقال الجديد",
    "description": "وصف المقال الجديد",
    "datePublished": "2025-08-02",
    "dateModified": "2025-08-02",
    "mainEntityOfPage": {
        "@type": "WebPage",
        "@id": "https://www.algmmal.com/new-article.html"
    },
    "articleSection": "فئة المقال",
    "keywords": "الكلمات المفتاحية مفصولة بفواصل"
}
```

### الخطوة 4: كتابة المحتوى

#### استخدام العناوين:
```html
<h2>عنوان رئيسي</h2>
<h3>عنوان فرعي</h3>
```

#### كتابة الفقرات:
```html
<p>
    محتوى الفقرة هنا. استخدم فقرات متعددة لتنظيم المحتوى بشكل أفضل.
</p>
```

#### إضافة القوائم:
```html
<ul>
    <li>النقطة الأولى</li>
    <li>النقطة الثانية</li>
    <li>النقطة الثالثة</li>
</ul>

<ol>
    <li>الخطوة الأولى</li>
    <li>الخطوة الثانية</li>
    <li>الخطوة الثالثة</li>
</ol>
```

#### إضافة الاقتباسات:
```html
<blockquote>
    هذا اقتباس مهم أو نقطة رئيسية تريد تسليط الضوء عليها.
</blockquote>
```

#### إضافة الكود:
```html
<!-- للكود المضمن -->
<code>console.log("Hello World");</code>

<!-- لكتل الكود -->
<pre><code>function sayHello() {
    console.log("مرحباً بالعالم!");
}

sayHello();</code></pre>
```

#### إضافة الصور:
```html
<img src="path/to/image.jpg" alt="وصف الصورة" class="article-image">
```

## أفضل الممارسات

### 1. كتابة المحتوى
- **العنوان**: يجب أن يكون واضحاً وجذاباً (50-60 حرف)
- **الوصف**: ملخص مفيد للمقال (150-160 حرف)
- **المقدمة**: اجذب القارئ من الجملة الأولى
- **التنظيم**: استخدم عناوين فرعية لتنظيم المحتوى
- **الخاتمة**: لخص النقاط الرئيسية وقدم دعوة للعمل

### 2. تحسين محركات البحث
- **الكلمات المفتاحية**: استخدمها بشكل طبيعي في المحتوى
- **الروابط الداخلية**: اربط بمقالات أخرى ذات صلة
- **النص البديل للصور**: اكتب وصفاً دقيقاً لكل صورة
- **URL**: استخدم رابطاً واضحاً ومفهوماً

### 3. تجربة المستخدم
- **طول المقال**: 800-2000 كلمة مثالي لمعظم المواضيع
- **الفقرات**: اجعلها قصيرة (2-3 جمل)
- **القوائم**: استخدمها لتنظيم المعلومات
- **الصور**: أضف صوراً ذات صلة لكسر النص

### 4. الأداء
- **حجم الصور**: ضغط الصور قبل الرفع
- **الكود**: تجنب الكود الثقيل غير الضروري
- **الخطوط**: استخدم خطوط محسنة للويب

## استكشاف الأخطاء

### المشاكل الشائعة وحلولها

#### 1. الإعلانات لا تظهر
- تأكد من تضمين `<script src="ads.js"></script>`
- تحقق من أن نظام الإعلانات مفعل في `ads.js`

#### 2. شريط التقدم لا يعمل
- تأكد من وجود عنصر `.article-content`
- تحقق من تضمين JavaScript الخاص بالشريط

#### 3. أزرار المشاركة لا تعمل
- تأكد من تضمين دوال المشاركة في JavaScript
- تحقق من صحة روابط المشاركة

#### 4. مشاكل التصميم المتجاوب
- اختبر على أجهزة مختلفة
- استخدم أدوات المطور في المتصفح
- تأكد من تضمين `viewport` meta tag

## مثال كامل

إليك مثال على مقال مكتمل:

```html
<!-- في قسم المحتوى -->
<article class="article-content">
    <h2>مقدمة عن JavaScript</h2>
    <p>
        JavaScript هي لغة برمجة قوية ومرنة تُستخدم في تطوير المواقع الحديثة.
        في هذا المقال، سنتعرف على أساسيات هذه اللغة الرائعة.
    </p>

    <h2>ما هو JavaScript؟</h2>
    <p>
        JavaScript هي لغة برمجة عالية المستوى تُستخدم لإضافة التفاعل 
        والديناميكية إلى صفحات الويب.
    </p>

    <h3>مميزات JavaScript</h3>
    <ul>
        <li>سهولة التعلم والاستخدام</li>
        <li>مرونة في التطبيق</li>
        <li>دعم واسع من المجتمع</li>
        <li>تطوير مستمر</li>
    </ul>

    <blockquote>
        "JavaScript هي لغة المستقبل في تطوير الويب"
    </blockquote>

    <h2>مثال بسيط</h2>
    <p>إليك مثال بسيط على كود JavaScript:</p>

    <pre><code>// طباعة رسالة ترحيب
function sayHello(name) {
    console.log(`مرحباً ${name}!`);
}

sayHello("محمد");</code></pre>

    <h2>خاتمة</h2>
    <p>
        JavaScript لغة قوية ومفيدة لكل مطور ويب. ابدأ رحلتك معها اليوم
        وستكتشف إمكانيات لا محدودة.
    </p>
</article>
```

**ملاحظة**: تأكد من اختبار المقال على أجهزة مختلفة قبل النشر، وتحقق من عمل جميع الروابط والميزات بشكل صحيح.
